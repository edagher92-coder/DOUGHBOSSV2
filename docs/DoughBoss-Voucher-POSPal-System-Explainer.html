How Vouchers, POSPal & Scanning Work

A plain-English walkthrough of the voucher system end to end — from a customer claiming a code, to it appearing as a real discount on the Revesby till.
Prepared by Elie D. for the Dough Boss ownership team  ·  1 July 2026  ·  Confidential

Short version: a customer claims a $5 or $10 voucher on the website. That code works two ways — spent online at checkout, or brought in-store and either scanned by staff or, at Revesby, applied automatically as a real discount at the till. Revesby's till connection is live and verified as of 1 July 2026.

Every step from claim to redemption is logged, and a code can only ever be spent once — online or in-store, not both.

2Step 1–2 — the customer-facing side (front end)

A customer claims a voucher through a widget on the website (the [doughboss_voucher_claim] block), entering their mobile number. The system:

  • Checks the daily pool hasn't run out (the $5 and $10 vouchers share one pool of 100 claims a day).
  • Generates a unique, typo-resistant code — built so a customer can't easily guess or mistype their way to a working code.
  • Displays it with a scannable QR code on screen, so it can be shown on a phone at the till.

That single code is now valid for exactly one use, online or in-store — whichever happens first.

3Step 3 — how it reaches the Revesby till (POSPal)

This is the part that connects the website to the physical POS system:

  • The moment a voucher is claimed with a phone number, the system automatically creates a matching coupon on the Revesby POSPal account, tied to that phone number.
  • When the voucher is later redeemed (online or in-store), the system tells POSPal to mark that coupon used — so it can never be spent a second time in either place.
  • This connection is authenticated and signed — every request to POSPal carries a signature built from a secret key that never leaves the server, so nothing can be faked or intercepted.
Status: live. Revesby's POSPal connection is switched on, and both the $5 and $10 discount rules are created and verified (confirmed 1 July 2026) — a live check found both rules and matched them correctly. The next voucher claimed with a phone number will grant automatically.

4Step 4 — redeeming it (two paths)

Path A — online, at checkout

The customer enters the code in the cart's voucher box. The discount is recalculated on the server (never trusted from the browser) and applied to the order. The system marks the code used the instant the order is placed — this also tells POSPal to cancel the matching till coupon, so it can't be reused in-store.

Path B — in-store, at Revesby

Two ways this can happen:

  1. Staff scan it — using the Voucher Scan tool (camera QR scan or manual code entry, in wp-admin or the standalone staff Console app) — this checks and redeems the code the same way as online.
  2. It's already sitting on the till as a POSPal coupon (from Step 3) — staff can apply it directly in POSPal against that customer's phone number, without needing to open the Scan tool at all.

5Where staff scanning fits in

The "Voucher Scan" tool is the staff-facing side of redemption — it exists in two places:

WhereWhat it's for
wp-admin "Voucher Scan" dashboardBuilt into the main WordPress admin, for staff who already have a WordPress login.
Standalone staff Console appA separate, install-on-any-device mini app (camera QR scan included) so a kitchen tablet doesn't need a full WordPress login — just enough access to scan vouchers, manage them, and watch the order board.

Both do the same underlying redemption — check the code, mark it used once, tell POSPal to cancel the matching till coupon.

6One important exception — vouchers made manually in WP admin

There's a separate "Voucher Maker" tool in WordPress admin for staff to create one-off codes by hand (for a promotion, a goodwill gesture, etc.). These do not automatically reach POSPal — only vouchers claimed by a customer through the website widget (Step 1) trigger the automatic POSPal grant. A hand-made code still works perfectly as an online discount, and can still be redeemed in-store via the Scan tool — it just won't already be sitting on the POSPal till as a coupon the way a customer-claimed one is. If staff need a code to be usable directly at the till without scanning, it needs to go through the normal claim flow, not the manual maker.

7What's still worth doing before relying on this for real customers

Recommended: one real end-to-end test. The check run so far confirms the connection and that the two coupon rules exist correctly — it hasn't yet exercised a real claim through to a granted till coupon. Before the first paying customer uses this:
  1. Claim a $5 or $10 voucher through the actual website with a real mobile number.
  2. Check the Revesby POSPal coupon/member list — the code should appear there within moments.
  3. Redeem it (online or via the Scan tool) and confirm it flips to used/cancelled in POSPal too.
