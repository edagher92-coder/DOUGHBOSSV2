2Executive Summary

The DoughBoss platform is built and running. What started as a commission-free ordering system for Dough Boss has grown into a genuinely full-featured operation: a branded storefront with a custom pizza builder, GST-correct pricing computed safely on the server, a real-time kitchen order board, a working voucher/discount engine, and the groundwork for card payments, catering enquiries, and a dedicated staff app — all already in the codebase, not on a whiteboard.

The build has been through sixteen separate hardening passes covering security, accessibility, and everyday usability — this isn't a first draft, it's a system that's been stress-tested against its own edge cases repeatedly. Per internal tracking, the entire effort to date has cost $600 AUD in AI-assisted development, covering both the backend plugin and the customer-facing storefront as one continuous build.

What's left is not a long list of engineering unknowns — it's a short, specific list of business decisions and a handful of details only Dough Boss can provide (menu, shop details, ABN, and a couple of POSPal back-office settings). Once those land, along with confirming which switched-off features to enable for day one, the plugin is in a position to soft-launch at a single configured shop.

5What's Left to Go Live

One blocking business item — the last link in the voucher loop

Vouchers can already be issued, scanned and marked redeemed in DoughBoss, and the code that grants and revokes a till discount on claim/redeem is already fully built and connected end-to-end. The only remaining step is Dough Boss creating the $5 and $10 discount rules inside the POSPal back office and giving us the resulting rule IDs to enter into settings. This is a setup task on your side, not a coding task on ours — once those two rule IDs are entered, the loop is live. (POSPal access is currently provisioned for one shop only; additional shops need their own POSPal account connected first.)

Business inputs we need from you

  • The real, final menu — item names, photos, AUD prices, pizza sizes and toppings, any combos.
  • Each shop's address, trading hours, pickup/delivery details and your ABN (for tax invoices).
  • Confirm the final brand name for launch materials — Dough Boss, with "Bite Pizzas" as a project working name, or something else — so we lock it in before anything goes to print.

Hardware & hosting readiness

  • A kitchen tablet or screen with reliable wifi at each shop going live.
  • WordPress/hosting login access, a fresh backup taken, and ideally moving off any free/basic hosting tier before launch.

Switch-on decisions already built and waiting

  • Card payments (Stripe) built · off — needs live keys and a go-ahead.
  • Multi-shop routing built · one shop configured — additional shops (e.g. a Revesby location) can be added in the admin as they're ready to go live.

6Recommended Next Step

Path to a first soft-launch:

  1. Lock the menu and shop details — send through final items, prices and the operating details for the first shop going live so we can load the real menu and configure that shop in the admin.
  2. Set the two POSPal discount rules for that shop and hand us the rule IDs — this closes the voucher-to-till loop.
  3. Decide on Stripe — tell us whether to switch it on for day one; it can also follow later with zero rebuild.
  4. Soft-launch the first shop — run it live at one shop first, confirm it end-to-end with real orders, then configure and roll out to additional shops with confidence.
The platform is built, hardened, and waiting. What's left is a short list of decisions only Dough Boss can make — once those are in, we're days, not months, from live orders.
